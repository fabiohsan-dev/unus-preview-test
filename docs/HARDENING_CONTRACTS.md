# UNUS Hardening Contracts

## Public `/venda` Query Contract

Canonical params:

- `tipo`
- `finalidade`
- `cidade`
- `bairro`
- `precoMin`
- `precoMax`
- `areaMin`
- `areaMax`
- `quartos`
- `suites`
- `vagas`
- `banheiros`
- `codigo`
- `ordem`
- `view`
- `page`

Legacy aliases are accepted only for compatibility:

- `categoria` -> `tipo`
- `negocio` -> `finalidade`

Server rendering redirects legacy or non-canonical query strings to the canonical URL. `/venda` without query is indexable; filtered URLs are `noindex, follow` and canonicalize to `/venda`.

## Property Slugs

Property slugs use `tipo-bairro-codigo`, generated by `src/lib/slug.ts`.

Old code-only or incomplete detail URLs are loaded by code, then redirected to the canonical slug when the Vista response is available.

## Favorites v2

Favorites in `localStorage` use `unus:favorites` with normalized ids:

- `imovel:CODIGO`
- `empreendimento:CODIGO`

The hook migrates legacy values automatically:

- `123` -> `imovel:123`
- `imovel-123` -> `imovel:123`
- `emp-123` -> `empreendimento:123`

## WhatsApp Leads

Lead messages are centralized in `src/lib/whatsapp.ts`. Helpers receive structured data and include origin/link context when available.

Current lead flow intentionally opens WhatsApp and does not add backend, database, webhook, or transactional email.

## Protected Images

Image protection is lightweight deterrence:

- disabled right-click on protected image elements
- `draggable={false}`
- `select-none`
- subtle `UNUS` CSS overlay

This is not technical DRM and does not prevent screenshots, browser cache access, network inspection, or determined downloads.

## Vista Credentials

Vista scripts and tests read `VISTA_BASE_URL` and `VISTA_KEY` from environment variables. Do not commit real keys.

The previously exposed Vista key must be rotated outside this repository and in Vercel/project secrets by someone with account access.
